Detailed Description

The XRayRequest class provides an interface for XRay requests.

Member Type Documentation

enum XRayRequest::Action
flags XRayRequest::Actions

This enum describes the actions that can be performed as XRay requests.

ConstantValueDescription
QtAws::XRay::XRayRequest::BatchGetTracesAction0XRay BatchGetTraces action.
QtAws::XRay::XRayRequest::GetServiceGraphAction1XRay GetServiceGraph action.
QtAws::XRay::XRayRequest::GetTraceGraphAction2XRay GetTraceGraph action.
QtAws::XRay::XRayRequest::GetTraceSummariesAction3XRay GetTraceSummaries action.
QtAws::XRay::XRayRequest::PutTelemetryRecordsAction4XRay PutTelemetryRecords action.
QtAws::XRay::XRayRequest::PutTraceSegmentsAction5XRay PutTraceSegments action.

The Actions type is a typedef for QFlags<Action>. It stores an OR combination of Action values.

[override virtual protected] QNetworkRequest XRayRequest::unsignedRequest(const QUrl &endpoint) const

Reimplemented from AwsAbstractRequest::unsignedRequest().

Returns a network request for the XRay request using the given endpoint.

This XRay implementation builds request URLs by combining the common query parameters (such as Action and Version), with any that have been added (via setParameter) by child classes.

XRayRequest &XRayRequest::operator=(const XRayRequest &other)

[virtual] bool XRayRequest::operator==(const XRayRequest &other) const

Returns true if this request is equal to other; false otherwise.

Note, most derived *Request classes do not need to provider their own implementations of this function, since most such request classes rely on this class' parameters functionality for all request parameters, and that parameters map is already checked via this implementation.
