Detailed Description

The MobileRequest class provides an interface for Mobile requests.

Member Type Documentation

enum MobileRequest::Action
flags MobileRequest::Actions

This enum describes the actions that can be performed as Mobile requests.

ConstantValueDescription
QtAws::Mobile::MobileRequest::CreateProjectAction0Mobile CreateProject action.
QtAws::Mobile::MobileRequest::DeleteProjectAction1Mobile DeleteProject action.
QtAws::Mobile::MobileRequest::DescribeBundleAction2Mobile DescribeBundle action.
QtAws::Mobile::MobileRequest::DescribeProjectAction3Mobile DescribeProject action.
QtAws::Mobile::MobileRequest::ExportBundleAction4Mobile ExportBundle action.
QtAws::Mobile::MobileRequest::ExportProjectAction5Mobile ExportProject action.
QtAws::Mobile::MobileRequest::ListBundlesAction6Mobile ListBundles action.
QtAws::Mobile::MobileRequest::ListProjectsAction7Mobile ListProjects action.
QtAws::Mobile::MobileRequest::UpdateProjectAction8Mobile UpdateProject action.

The Actions type is a typedef for QFlags<Action>. It stores an OR combination of Action values.

[override virtual protected] QNetworkRequest MobileRequest::unsignedRequest(const QUrl &endpoint) const

Reimplemented from AwsAbstractRequest::unsignedRequest().

Returns a network request for the Mobile request using the given endpoint.

This Mobile implementation builds request URLs by combining the common query parameters (such as Action and Version), with any that have been added (via setParameter) by child classes.

MobileRequest &MobileRequest::operator=(const MobileRequest &other)

[virtual] bool MobileRequest::operator==(const MobileRequest &other) const

Returns true if this request is equal to other; false otherwise.

Note, most derived *Request classes do not need to provider their own implementations of this function, since most such request classes rely on this class' parameters functionality for all request parameters, and that parameters map is already checked via this implementation.
